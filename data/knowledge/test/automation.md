# 自动化测试

## 接口自动化

接口测试关注点：

- 请求参数校验（必填、类型、边界）
- 响应状态码和数据结构
- 异常场景（超时、鉴权失败、参数错误）

常用工具：Postman、JMeter、REST Assured。

## UI 自动化

UI 自动化框架选型：

- **Web**：Selenium、Playwright、Cypress
- **移动端**：Appium

最佳实践：

- 使用 Page Object 模式分离定位与业务
- 减少硬等待，用显式等待
- 测试数据与脚本分离

## 性能测试

性能测试类型：

- 负载测试：正常压力下系统表现
- 压力测试：超出预期的极限压力
- 稳定性测试：长时间运行是否内存泄漏

关键指标：TPS、响应时间（P95/P99）、错误率、资源利用率。

## CI/CD 集成

自动化测试接入流水线：

1. 代码提交触发单元测试
2. 合并前运行接口测试
3. 发布前执行冒烟测试和回归测试
