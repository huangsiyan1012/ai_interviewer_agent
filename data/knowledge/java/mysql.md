# MySQL 数据库

## 索引原理

MySQL InnoDB 使用 B+ 树存储索引：

- 聚簇索引：叶子节点存储完整行数据
- 二级索引：叶子节点存储主键值，需要回表查询

## 索引失效场景

以下情况可能导致索引失效：

1. 对索引列使用函数或运算：`WHERE YEAR(create_time) = 2024`
2. 隐式类型转换：`WHERE phone = 13800138000`（phone 是 varchar）
3. 左模糊查询：`WHERE name LIKE '%张'`
4. OR 条件一侧无索引
5. 违反最左前缀原则（联合索引）

## SQL 优化

- 用 `EXPLAIN` 分析执行计划
- 避免 `SELECT *`，只查需要的列
- 大表分页用延迟关联或游标方式
- 合理使用覆盖索引减少回表

## 事务与锁

- 事务 ACID：原子性、一致性、隔离性、持久性
- 隔离级别：读未提交、读已提交、可重复读、串行化
- InnoDB 默认可重复读，通过 MVCC + 间隙锁解决幻读
